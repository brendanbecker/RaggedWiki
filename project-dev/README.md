# Project Development Files

This directory contains development artifacts and planning documents used during the creation of RaggedWiki.

## Files

### PLANNING.md
Original planning document outlining the project structure, goals, and implementation approach.

### IMPLEMENTATION_PROMPTS.md
Prompts and guidance used during the implementation of various project components.

### validate_phrase_similarity.py
Python script used to validate and test phrase similarity embeddings during development.
Used to empirically determine similarity score thresholds and embedding quality.

## Purpose

These files are kept for:
- Historical reference
- Understanding design decisions
- Reproducing development processes
- Validating technical approaches

## For Users

If you're learning from RaggedWiki, you don't need these files. Start with:
- **[../QUICKSTART.md](../QUICKSTART.md)** for hands-on quick start
- **[../docs/README.md](../docs/README.md)** for the curriculum
- **[../simple_rag_system/TESTING.md](../simple_rag_system/TESTING.md)** to verify the RAG system
